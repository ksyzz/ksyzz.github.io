前言

今天在IDEA上写了一个项目，使用IDEA运行main函数时，程序运行正常，但是放到服务器上，使用maven的指令：mvn exec:java -Dexec.mainClass=””的时候，报了一个八竿子打不着的错误：ObjectInputStream的异常。
但是项目在IDEA上执行是正常的，于是我在本地也使用mvn exec来执行项目，发现还是报错。
陷入了沉思。。。

解决思路

IDEA运行时，使用的class文件和mvn exec使用的文件是一致的，那么问题就应该出现在依赖的包上，于是写了另一个Main类，在里面调用了引用的依赖中的一个类，然后使用mvc exec执行这个Main类，发现报错ClassNotFoundExeception。

问题就很明显了，mvn exec执行的时候，并没有将依赖引入进来。

解决方式

考虑到maven引入依赖的时候，所有依赖默认的scope都是compile。
exec文档说可以修改执行时类路径的scope，于是在原有指令后面加了一个 -Dexec.classpathScope=compile，程序就能正常运行了。
